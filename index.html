Introduction

The Restaurant API allows the access and management of restaurant data including adding, deleting, updating.

Authentication

To use this API, you need to authenticate using a token. Pass the token in your requests.

GET hhttps://restaurantms.azurewebsites.net/category     
             Authorization: token

Endpoints

GET restaurants

Retrieve a list of restaurants.

GET https://restaurantms.azurewebsites.net/restaurants
 Authentication: token
Response:
[
{
    "id": 12345,
    "name": "Sarova",
    "street_address": "Naka01",

    "created_at": "2024-06-05T10:00:00.000Z",

]

POST

Adding a new city.

POST https://restaurantms.azurewebsites.net/city
Authentication: token
  Content-Type: application/json
{
    "name": "Kisumu",
    "state_id": 1
  }
Response:
{
    "City created successfully"
}

DELETE

Deleting a driver with ID 1.

DELETE https://restaurantms.azurewebsites.net/driver/1
  Authorization: token

PUT

Updating the driver with ID 1.

PUT https://restaurantms.azurewebsites.net/driver/1
  Authorization: token
  Content-Type: application/json
  {
    "car_make": "Toyota",
    "car_model": "Corolla",
    "car_year": 2020,
    "user_id": 1,
    "online": true,
    "delivering": false
  }
Response:
{
    "Driver updated successfully
}

Error Codes

Common error codes returned by the API.

  • 400 Bad Request: The request could not be understood or was missing required parameters.
  • 404 Not Found: The requested resource could not be found.
  • 500 Internal Server Error: An error occurred on the server.
